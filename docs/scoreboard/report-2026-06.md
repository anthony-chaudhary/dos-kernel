# Drift-rate scoreboard — aggregate report #1 (June 2026)

> Across **12 public repositories with machine-attributable agent commits**,
> **1.2%** of concrete claims made by agent-authored commit subjects are
> unwitnessed by the commit's own diff — and hand-adjudication of every fire
> attributes most of that residue to subject-style noise, not over-claiming.
> The unambiguous forgery shape — a concrete "fix …" claim on a commit that
> touched **nothing** — is 3 commits in 1,715 checkable claims (**0.17%**).

## Every fire, hand-adjudicated

The methodology promises the rate is inspectable, never just a number. The
21 fires, de-identified, by class:

- **9 — prose-scoped documentation edits.** An effect verb over a doc-only
  diff where the subject itself names the doc target in prose ("Update …
  in README", "Add … guide"). Honest doc work whose subject grammar reads
  as a code claim; would abstain or pass under a `docs:` head.
- **5 — test claims with net-deleted test lines.** Removals and migrations
  ("Remove the test tools", "Update tests") where deletion plausibly *is*
  the claimed work. The delete-the-assertion guard is deliberately strict;
  these are its known cost.
- **4 — test-noun grammar artifacts.** "test" as an adjective or inside a
  command name ("test repo URLs", a `/test-…` workflow), and a test-harness
  setup file living outside the test-path grammar.
- **3 — concrete fix claims on empty commits.** The canonical subject-only
  shape: the message claims a specific fix; the commit touched no files.

So the honest reading is **not** "agents lie 1.2% of the time." It is: in
established, attribution-honest, human-reviewed repositories, agent-authored
commit subjects are overwhelmingly witnessed by their own diffs — and the
witness still finds a small, real residue of claims that rest on message
text alone, including three textbook empty over-claims. Selection caveats
(visible attribution only, reviewed repos, English-grammar claims) are in
methodology §4 and §7.

## Witness-grammar fixes this run forced (published before the number)

Running the witness at corpus scale surfaced three over-fire classes, each
fixed by narrowing the fire before this report was published:
[#79](https://github.com/anthony-chaudhary/dos-kernel/issues/79) (doc head
vs. test noun), [#81](https://github.com/anthony-chaudhary/dos-kernel/issues/81)
(config/data diffs — 98 of 99 flags in the pilot),
[#94](https://github.com/anthony-chaudhary/dos-kernel/issues/94) (`SKILL.md`
is the program of an agent system — 66 of 88 raw flags in the first full
sweep). Raw pooled drift before those fixes read 5.1%; the adjudicated
truth did not change, the grader did.

## Reproduce

```bash
python scripts/drift_scoreboard.py --corpus corpus.txt --out out/ \
    --scan-limit 10000 --audit-limit 300 --stamp 2026-06-12
```

Or score your own repo in one command:
`dos commit-audit --sweep --workspace . BASE..HEAD`.
